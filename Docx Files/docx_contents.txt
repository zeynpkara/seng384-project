Regular commits are expected to be pushed to the GitHub repository at every meaningful development step.
Project Flow
1. STEP : Frontend + SRS
Screens are completed. Not just the design — the frontend must be able to handle at least mock requests coming from the backend 
2. STEP : Full Stack Integration + SDD
The SDD document is prepared; database and web service connections are completed. The core skeleton of the application takes shape. A complete backend–data–frontend flow must be established.
3. STEP : Final Submission + User Guide + Presentation Video (max. 5 minutes)
The project is finalized. A project report is created by incorporating a User Guide alongside the updated versions of the SRS and SDD documents, and the demo video is uploaded.
